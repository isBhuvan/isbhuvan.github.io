# Bhuvaneshwaran | Personal Portfolio

A clean, responsive, and modern portfolio website designed to showcase my work, skills, and experience as a **Cloud Engineer**. This repository contains the source code for my personal portfolio site hosted via GitHub Pages.

The site highlights:

- Cloud architecture projects
- Infrastructure as Code implementations
- Automation tooling in Go
- Certification preparation journeys
- Engineering philosophy and reliability practices

---

## 🌐 Live Site

You can view the live version of this portfolio here: [https://isbhuvan.github.io/](https://isbhuvan.github.io/)

---

## ✨ Features

- **Responsive Design:** Fully optimized for mobile, tablet, and desktop using grid system.
- **Project Gallery:** Interactive cards showcasing my latest work with links to GitHub and live demos.
- **Modern UI:** Custom CSS animations and a clean aesthetic.
- **SEO Optimized:** Proper use of semantic HTML5 tags.

---

## 🛠️ Tech Stack

- **HTML5:** Structure and Semantics.
- **CSS3:** Custom styling and layout.
- **JavaScript (ES6):** Interactivity and dynamic elements.
- **Google Fonts:** Typography.

---

## 📂 Repository Structure

```text
├── assets/
│   ├── css/      # Custom styles

│   └── images/      # Optimized project images and icons

```

## 🚀 Release Process

This project follows:

- Conventional Commits
- Semantic Versioning
- Automated CHANGELOG generation
- GitHub Actions-based release workflow

Each merge to `main` may trigger a version bump and changelog update.

## 🔍 Quality Controls

- Markdown linting (`make lint-md`)
- Commit lint enforcement (`make lint-commit`)
- HTML linting (`make lint-html`)
- Lighthouse CI checks (`make lighthouse`)
- Aggregated local checks (`make check`)
- Pull request quality checks (`make pr-check`)

## ⚙️ Installation & Setup

To run this project locally:

1. **Clone the repository:**

    ```bash
    git clone https://github.com/isbhuvan/isbhuvan.github.io.git
    ```

2. **Navigate to the directory:**

    ```bash
    cd isbhuvan.github.io
    ```

3. **Open in your browser:**
    Simply open `index.html` in your favorite browser or use the **Live Server** extension in VS Code.
